## 第一阶段：Python 筑基与工程习惯（4~5 周）

> **目标**：能独立写出带测试的命令行工具，所有代码用 Git 管理。
> **里程碑**：用 Python + Git + pytest 完成一个“终端版待办事项管理器”。

### 1.1 编程核心

| 知识点 | 优先级 | 具体内容 | 掌握标准 |
|--------|--------|----------|----------|
| 变量与基本数据类型 | ⭐⭐⭐ | int、float、str、bool | 能区分数字/字符串/布尔值的适用场景 |
| 字符串操作 | ⭐⭐⭐ | 切片、f-string、join、split、strip、replace | 能独立处理格式化输出和简单文本解析 |
| 列表与元组 | ⭐⭐⭐ | 增删改查、切片、排序、列表推导式、元组不可变性 | 能选择合适的结构存储数据，熟练使用推导式 |
| 字典与集合 | ⭐⭐⭐ | 增删改查、keys/values/items、集合去重与运算 | 能随时用字典组织配置或键值数据 |
| 控制流 | ⭐⭐⭐ | if/elif/else、for、while、break、continue、pass | 能写出清晰的嵌套逻辑，不绕晕自己 |
| 函数 | ⭐⭐⭐ | 定义、参数（位置/关键字/默认值）、返回值、lambda | 能写职责单一的函数，习惯用函数组织代码 |
| 作用域基础 | ⭐⭐ | 局部变量、全局变量、LEGB 规则（先了解，不钻 nonlocal） | 知道变量在哪里能用，避免意外覆盖 |
| 异常处理 | ⭐⭐⭐ | try/except/else/finally、raise 简单异常 | 程序不会因为一个小错误就崩溃，知道如何给出清晰提示 |
| 文件操作 | ⭐⭐⭐ | open、with 上下文、读/写文本、JSON 序列化 | 能读写配置文件、保存任务列表到文件 |
| 模块与包 | ⭐⭐⭐ | import、from...import、`__name__`、包的基本结构 | 能把代码拆成多个文件管理 |

### 1.3 立刻开始的工程习惯

| 知识点 | 优先级 | 具体内容 | 掌握标准 |
|--------|--------|----------|----------|

## 第二阶段：Web 入门与第一个 API 项目（4~5 周）

> **目标**：理解关系型数据库，做出一个带持久化存储的 REST API，并运行在本地 Docker 中。
> **里程碑**：完成“Todo API”或“图书管理 API”，支持用户登录、CRUD 操作，用 Docker Compose 一键启动。

### 2.1 数据库（SQL + MySQL 或其他关系型数据库）—— 合并学习，直接用真实数据库练习

| 知识点 | 优先级 | 具体内容 | 掌握标准 |
|--------|--------|----------|----------|
| 数据库核心概念 | ⭐⭐⭐ | 表、字段、记录、主键、外键、索引 | 能设计简单的 ER 图，知道为什么需要主键和外键 |
| 数据类型 | ⭐⭐⭐ | INT、VARCHAR、TEXT、BOOLEAN、DATE、DATETIME | 创建表时能根据业务选对类型 |
| 查询基础 | ⭐⭐⭐ | SELECT、FROM、WHERE、AND/OR、IN、BETWEEN、LIKE、LIMIT | 能手写常见的过滤查询，不用依赖图形化工具 |
| 排序与分页 | ⭐⭐⭐ | ORDER BY、LIMIT/OFFSET | 能够实现列表页的后端查询 |
| 聚合与分组 | ⭐⭐⭐ | COUNT、SUM、AVG、GROUP BY、HAVING | 能回答“某分类下有多少条数据”这类统计问题 |
| 表连接 | ⭐⭐⭐ | INNER JOIN、LEFT JOIN | 能凭查询拉出关联表的数据 |
| 数据修改 | ⭐⭐⭐ | INSERT、UPDATE、DELETE | 能正确执行并注意 WHERE 条件 |
| 约束 | ⭐⭐⭐ | NOT NULL、UNIQUE、PRIMARY KEY、FOREIGN KEY | 表的创建语句中能写出全部业务约束 |
| 事务 | ⭐⭐ | BEGIN/COMMIT/ROLLBACK | 知道“一荣俱荣，一损俱损”的场景，例如转账 |
| SQL 注入与防护 | ⭐⭐⭐ | 参数化查询的本质 | 永远不拼接字符串去执行 SQL |
| 命令行与工具 | ⭐⭐ | 用命令行或 GUI 连接数据库，执行 DDL/DML | 能脱离代码直接操作数据库进行调试 |
| 数据库与驱动 | ⭐⭐⭐ | 安装 MySQL/PostgreSQL，选用对应的异步 Python 驱动（如 asyncmy 或 asyncpg） | 开发环境能随时起一个数据库实例 |

### 2.2 FastAPI 核心

| 知识点 | 优先级 | 具体内容 | 掌握标准 |
|--------|--------|----------|----------|
| 应用创建与启动 | ⭐⭐⭐ | `FastAPI()`、`uvicorn` 运行、自动文档 `/docs` | 会从零开始创建一个服务并访问文档 |
| 路由与 HTTP 方法 | ⭐⭐⭐ | `@app.get/post/put/delete`、路径参数、查询参数 | 能设计出符合 RESTful 风格的接口 |
| 请求体与 Pydantic | ⭐⭐⭐ | `BaseModel`、字段类型、必填/可选/默认值、嵌套模型 | 用模型清晰定义输入输出，享受自动校验的便利 |
| 字段校验 | ⭐⭐⭐ | `Field(min_length=..., ge=..., pattern=...)` | 不用再在函数里写一堆 if 判断 |
| 响应模型 | ⭐⭐⭐ | `response_model`、状态码 `status_code` | 接口输出的字段完全可控，不会暴露多余数据 |
| 依赖注入（重点） | ⭐⭐⭐ | `Depends()`、数据库会话依赖、`yield` 管理资源 | 能封装重复逻辑（如提取当前用户、获取数据库会话），让路由函数保持简洁 |
| ORM 集成 (SQLAlchemy 2.0) | ⭐⭐⭐ | 异步 Engine、异步 Session、模型定义、关系映射 | 能用 ORM 替代手写 SQL 进行 CRUD，并明白背后生成的 SQL 是什么 |
| 数据库迁移 | ⭐⭐ | Alembic 初始化、生成迁移、升级/降级 | 表结构变更能够版本控制，不再改了就慌 |
| 异常处理 | ⭐⭐ | `HTTPException` 基本用法 | 能返回语义正确的错误响应（如 404、401） |
| 配置管理 | ⭐⭐⭐ | `pydantic-settings` 或 `python-dotenv`，环境变量读取数据库地址、密钥等 | 敏感信息不会提交进代码仓库 |

### 2.3 容器化基础

| 知识点 | 优先级 | 具体内容 | 掌握标准 |
|--------|--------|----------|----------|
| Docker 核心概念 | ⭐⭐⭐ | 镜像、容器、Dockerfile、docker-compose | 能解释为什么“我这能跑”而“你那不行” |
| 编写 Dockerfile | ⭐⭐⭐ | FROM、WORKDIR、COPY、RUN、CMD | 能把 FastAPI 项目打包成镜像 |
| Docker Compose 编排 | ⭐⭐⭐ | 定义 FastAPI + 数据库 + (可选) Redis 服务 | 一条命令启动整个开发环境 |
| 基本操作 | ⭐⭐⭐ | `docker build`, `docker run`, `docker compose up` | 熟练使用，不再抗拒命令行 |

### 2.4 第一个小项目

用上面所有知识，做出一个简单的 **Todo API** 或 **图书管理 API**，要求：

- 用户注册、登录（先简单用哈希密码 + API Key，暂时不引入 JWT）
- 待办事项的增删改查（CRUD）
- 列表支持分页
- 全部接口用 pytest + TestClient 写好测试
- 数据库用异步 SQLAlchemy + Alembic 管理
- 最终用 `docker compose up` 就能运行

## 第三阶段：工程化与高级特性（4~5 周）

> **目标**：补完 Python 异步/高级语法，加入安全认证、中间件、生产部署，交付完整的博客系统。
> **里程碑**：部署上线一个真正的博客系统，有 JWT 认证、CI（Github Actions 跑测试）、HTTPS 加反向代理。

### 3.1 Python 进阶与异步

| 知识点 | 优先级 | 具体内容 | 掌握标准 |
|--------|--------|----------|----------|
| 类型注解进阶 | ⭐⭐⭐ | `Optional`、`Union`、`List[xxx]`、`Dict`、`Any` | 代码里的所有函数参数和返回值都有清晰类型 |
| 装饰器 | ⭐⭐⭐ | 函数装饰器、带参数装饰器、保留元数据 `@wraps` | 能自己写日志装饰器、权限校验装饰器 |
| 迭代器与生成器 | ⭐⭐ | `yield`、`iter`、生成器表达式 | 理解惰性计算，知道何时用生成器节约内存 |
| 异步编程 | ⭐⭐⭐ | `async/await`、`asyncio.run`、`create_task`、`gather` | 能写出并发执行的数据库查询或外部 API 调用 |
| 上下文管理器进阶 | ⭐⭐ | `contextlib.contextmanager`、`asynccontextmanager` | 能自己封装资源管理的 with 块 |

### 3.2 安全与中间件

| 知识点 | 优先级 | 具体内容 | 掌握标准 |
|--------|--------|----------|----------|
| 密码哈希 | ⭐⭐⭐ | bcrypt / passlib 哈希与验证 | 数据库中绝不存明文密码 |
| JWT 认证 | ⭐⭐⭐ | 生成 Token、验证、过期刷新、提取当前用户依赖 | 能实现完全无状态的用户认证 |
| OAuth2 集成 | ⭐⭐⭐ | FastAPI 的 `OAuth2PasswordBearer`、`OAuth2PasswordRequestForm` | 接口线自觉支持标准认证流程 |
| 权限控制 | ⭐⭐ | 基于角色的简单权限（admin、user） | 能区分普通用户和管理员接口 |
| CORS 配置 | ⭐⭐⭐ | `CORSMiddleware` 设置 origins、methods、headers | 前后端分离项目不报跨域错误 |
| 自定义中间件 | ⭐⭐⭐ | 记录请求耗时、添加请求 ID、日志打印 | 每个请求都有迹可循 |
| 后台任务 | ⭐⭐ | `BackgroundTasks` 发送邮件、写审计日志等 | 接口响应速度不被慢操作拖累 |

### 3.3 测试与质量

| 知识点 | 优先级 | 具体内容 | 掌握标准 |
|--------|--------|----------|----------|
| 依赖覆盖 | ⭐⭐⭐ | `app.dependency_overrides` 替换数据库会话等 | 测试不依赖真实数据库 |
| TestClient 进阶 | ⭐⭐⭐ | 带上认证头、构造各类请求体 | 能在测试中模拟真实用户操作 |
| 参数化测试 | ⭐⭐ | `@pytest.mark.parametrize` | 一组用例覆盖多种边界情况，代码更简洁 |
| 覆盖率 | ⭐⭐ | pytest-cov 查看哪些代码没测到 | 测试不再盲目，知道往哪里补 |

### 3.4 部署与运维

| 知识点 | 优先级 | 具体内容 | 掌握标准 |
|--------|--------|----------|----------|
| Linux 基础操作 | ⭐⭐⭐ | 文件操作、权限、进程查看、curl、ssh | 能在云服务器上独立完成部署 |
| Gunicorn + Uvicorn | ⭐⭐⭐ | 多 worker 启动，绑定地址，生产配置 | 应用不再用 `uvicorn main:app` 裸奔 |
| Nginx 反向代理 | ⭐⭐⭐ | 配置代理转发、静态文件、隐藏服务端口 | 域名 80/443 端口正确转发到 FastAPI |
| SSL/HTTPS | ⭐⭐ | certbot 申请免费证书，Nginx 配置 443 | 浏览器地址栏出现小锁 |
| 进程守护 | ⭐⭐ | systemd 或 supervisor 保证挂了能重新拉起 | 服务不会因为退出终端而停止 |
| 日志管理 | ⭐⭐ | 日志输出格式、级别、logrotate 轮转 | 磁盘不会被几个月前的日志撑满 |
| CI/CD 初步 | ⭐⭐ | GitHub Actions 自动运行测试、构建镜像 | 提交代码就有机器人告诉你测试过没过 |

### 3.5 完整项目：博客系统

综合运用一二三阶段所有技能，实现一个包含以下功能的博客平台：

- 用户注册、登录、个人信息管理（JWT + 权限）
- 文章的发布、编辑、删除（Markdown 支持）
- 文章列表分页、搜索
- 评论功能、点赞（多对多关系）
- 后台管理接口
- 异步 ORM 查询优化（预加载、避免 N+1）
- 全部接口测试覆盖率 > 80%
- Docker Compose 一键启动开发环境
- 通过 GitHub Actions 自动测试，并最终部署到云服务器（Gunicorn + Nginx + PostgreSQL）

## 第四阶段：深化与拓展（按需，不设固定时长）

| 知识点 | 优先级 | 具体内容 | 学习时机 |
|--------|--------|----------|----------|
| 分布式任务队列 | 🔧 | Celery + Redis 处理耗时任务，定时任务 beat | 当接口响应时间因为发邮件、生成报表而变慢时 |
| WebSocket | 🔧 | 实时通知、聊天 | 业务需要长连接推送时 |
| 缓存 | 🔧 | Redis 缓存高频查询结果 | 数据库扛不住频繁相同的查询时 |
| 性能分析 | 🔧 | profiling、异步并发调优、数据库索引优化 | 接口在高并发下出现延迟毛刺 |
| 监控告警 | 🔧 | Prometheus + Grafana 基础监控，Sentry 错误追踪 | 线上系统用户量上来后 |
| GraphQL | 📅 | Strawberry + FastAPI | 需要前端灵活控制返回字段时 |
| Kubernetes | 📅 | 容器编排、自动伸缩 | 需要管理大量微服务实例时 |
| 设计模式 | 🔧 | 工厂、单例、策略等在项目里的实际应用 | 发现代码复用得越来越难受时 |
